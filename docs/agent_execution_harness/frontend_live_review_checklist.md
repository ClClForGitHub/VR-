# Frontend Live Review Checklist

Use this checklist after each Round 04 live case.

## API Snapshot

- [ ] `GET /api/runs/<run_key>` returns current phase/status.
- [ ] Concept requirements are visible.
- [ ] Asset library is visible.
- [ ] Active assembly selection is visible.
- [ ] File manifest includes concept images, model assets, scene assets, viewer, preview, and reports when available.
- [ ] Available actions match the current gate.

## User Gates

- [ ] Concept approval/rejection can be represented as backend action payload.
- [ ] Model approval/rejection can be represented as backend action payload.
- [ ] Asset selection can be represented as backend action payload.
- [ ] Blender preview approval/rejection can be represented as backend action payload.

## Visual Evidence

- [ ] Concept image thumbnails or URLs are accessible.
- [ ] Subject GLB/model links are accessible.
- [ ] Scene asset links are accessible.
- [ ] Final preview image is accessible.
- [ ] Viewer GLB URL works or records exact failure.
