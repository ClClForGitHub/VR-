# Round 01：核心 Harness 文档执行包

本 ZIP 是给 Codex / coding agent 的第一轮执行包。它不是功能开发包，而是建立后续所有任务的执行规程。

## 本轮目标

让仓库里出现一套短、硬、可引用的执行 harness，约束后续 agent：

- 按真实业务状态推进，不用“补 scaffold + 测试 passed”交差。
- 每轮都说明入口、输出、测试、前端状态和文档维护。
- 后续涉及模型服务时，优先做真实测试，但必须有明确命令边界、输出路径和用户批准。
- 用户聊天窗口里的概念图、模型、场景、预览都要逐步进入资产库设计，下一轮单独落地。

## 如何使用

1. 把 `01_给_Codex_执行包.md` 的内容直接发给 Codex。
2. 把 `repo_files_to_copy/` 整个目录也提供给 Codex，告诉它可以复制/改写里面的文件到仓库根目录。
3. 让 Codex 按执行包完成后，填写 `02_Codex_完成汇报模板.md`，并把结果贴回给你。
4. 你把 Codex 的汇报、测试输出、diff 或 commit 信息发给我，我判断是否合格，再给 Round 02。

## 本轮不做什么

- 不改 runtime 业务代码。
- 不改前端 UI。
- 不提交 live Hunyuan3D / HY-World / image generation 任务。
- 不生成真实资产。

本轮只建立规则、模板、文档索引和一个文档存在性测试。
